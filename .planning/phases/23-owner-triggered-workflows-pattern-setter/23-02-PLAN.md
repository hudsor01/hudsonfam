<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client component → Server Action | `requireRole(["owner"])` is the first call; non-owner redirect happens before any DB access or webhook fire |
| Server Action → webhooks.ts | `sendSignedWebhook` returns discriminated union; Server Action translates failure to `{ ok: false, sentinel }` — raw error objects never cross this return path |
| Server Action → DB (getJobDetail) | Read-only — reads `cover_letters.generated_at` before firing webhook; DB failure returns `{ ok: false, sentinel: "unavailable" }` via the same sentinel pattern |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-02-01 | Elevation of Privilege | triggerCompanyResearch / regenerateCoverLetter | mitigate | `await requireRole(["owner"])` as FIRST line (D-12); D-12 grep test in Plan 23-04 asserts this at CI level; any missing call is a test failure |
| T-23-02-02 | Information Disclosure | regenerateCoverLetter return value | mitigate | Baseline is `cover_letters.generated_at` ISO string — not sensitive PII; returned only to authenticated owner; `{ ok: false, sentinel }` on DB failure keeps error details server-side |
| T-23-02-03 | Repudiation | triggerCompanyResearch / regenerateCoverLetter idempotency | mitigate | `randomUUID()` per call (D-03) injected into `sendSignedWebhook`; test asserts two consecutive calls produce different UUIDs, preventing stale-key replay of the same run |
| T-23-02-04 | Denial of Service | Client triggering rapid Server Action calls | accept | Button state machine in Plans 23-05/23-06 disables the button during polling; no server-side rate gate needed at this layer (n8n deduplication + 5s AbortSignal.timeout in webhooks.ts provide backstop) |
| T-23-02-05 | Information Disclosure | Raw error from getJobDetail leak | mitigate | DB call wrapped in try/catch; on error returns `{ ok: false, sentinel: "unavailable" }` — no e.message in return value |
</threat_model>

    ```typescript
    /**
     * Owner-triggered: fire the job-company-intel n8n workflow for a job
     * whose company_research row is currently null.
     *
     * Requires owner role (D-12). Returns discriminated union — never throws (D-08).
     * Idempotency key is a fresh UUID per call (D-03).
     */
    export async function triggerCompanyResearch(
      jobId: number
    ): Promise<{ ok: true } | { ok: false; sentinel: ErrorSentinel }> {
      await requireRole(["owner"]);
      const idempotencyKey = randomUUID();
      const res = await sendSignedWebhook(
        "job-company-intel",
        { job_id: jobId },
        idempotencyKey
      );
      if (!res.ok) return { ok: false, sentinel: res.sentinel };
      revalidatePath("/admin/jobs");
      return { ok: true };
    }

    /**
     * Owner-triggered: fire the regenerate-cover-letter n8n workflow.
     * Reads pre-webhook cover_letters.generated_at from DB and returns it as
     * `baseline` so the client can poll until generated_at advances past
     * the baseline — no client-clock access needed (D-06 amended).
     *
     * Requires owner role (D-12). Returns discriminated union — never throws (D-08).
     */
    export async function regenerateCoverLetter(
      jobId: number
    ): Promise<
      | { ok: true; baseline: string | null }
      | { ok: false; sentinel: ErrorSentinel }
    > {
      await requireRole(["owner"]);
      let baseline: string | null = null;
      try {
        const detail = await getJobDetail(jobId);
        baseline = detail?.cover_letter?.generated_at ?? null;
      } catch {
        return { ok: false, sentinel: "unavailable" };
      }
      const idempotencyKey = randomUUID();
      const res = await sendSignedWebhook(
        "regenerate-cover-letter",
        { job_id: jobId },
        idempotencyKey
      );
      if (!res.ok) return { ok: false, sentinel: res.sentinel };
      revalidatePath("/admin/jobs");
      return { ok: true, baseline };
    }
    ```
