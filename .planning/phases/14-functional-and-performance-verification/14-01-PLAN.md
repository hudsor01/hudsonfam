  <name>Task 1: PERF-01 — Measure server-side data fetch time</name>
  <files>N/A (measurement only)</files>
  <action>
Measure the server-side response time for the jobs page:

```bash
# Time the full page load (includes SSR data fetch)
curl -sk -o /dev/null -w "time_total: %{time_total}s\ntime_starttransfer: %{time_starttransfer}s\n" https://thehudsonfam.com/admin/jobs
```

The `time_starttransfer` (TTFB) reflects the server-side data fetch time. Run 3 times and take the median.

Requirement: PERF-01 — server-side data fetch completes in under 2 seconds.

Note: This will return 307 (redirect to login) for unauthenticated requests. Use the redirect timing as a baseline, then verify actual page load via browser automation in Task 2.
  </action>
  <verify>
    <automated>curl -sk -o /dev/null -w "%{time_starttransfer}" https://thehudsonfam.com/admin/jobs | python3 -c "import sys; t=float(sys.stdin.read()); print(f'{t:.3f}s'); assert t < 2.0, f'TTFB {t:.3f}s exceeds 2s'"</automated>
  </verify>
  <done>TTFB measured 3 times, median is under 2 seconds.</done>
</task>

  <how-to-verify>
Use Chrome browser automation tools to verify each functional requirement against production.

**Step 1: Login and navigate to /admin/jobs**
- Navigate to https://thehudsonfam.com/login
- Login via Google OAuth (owner account)
- Navigate to https://thehudsonfam.com/admin/jobs
- Confirm page loads without JavaScript errors

**Step 2: FUNC-01 — Table view with sort/filter/pagination**
- Confirm table view is the default view
- Click a column header to sort — verify rows reorder
- Verify pagination controls are visible (if enough rows)
- Take note of job count

**Step 3: FUNC-02 — Kanban view with 6-stage pipeline**
- Switch to kanban view
- Verify 6 columns are visible: new, interested, applied, interview, offer, rejected
- Verify job cards are present in at least one column
- PERF-02: Verify no layout shift during render and drag handles are responsive

**Step 4: FUNC-03 — Kanban drag-drop persists**
- Drag a job card from one column to another (e.g., "new" → "interested")
- Note which job was moved and the target column
- Refresh the page (F5)
- Verify the card is still in the target column after refresh

**Step 5: FUNC-04 — Job detail sheet + PERF-03**
- Click on a job card or row to open the detail sheet
- Verify cover letter content is displayed
- Verify company intel/research section is displayed
- PERF-03: Measure time from click to full content display — should be under 1 second

**Step 6: FUNC-05 — Filter by source, status, and score range**
- Apply a source filter (pick any available source)
- Verify only jobs from that source are visible
- Clear filter, apply a status filter
- Verify only jobs with that status are visible
- Clear filter, apply a score range filter
- Verify visible jobs fall within the score range

**Step 7: FUNC-06 — Dismiss and restore**
- Find a job in the active view
- Dismiss it (use the dismiss action)
- Verify it disappears from the active view
- Navigate to dismissed view / restore the job
- Verify it reappears in the active view
- Refresh to confirm persistence
