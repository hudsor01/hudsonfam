| Token | Value | Usage in Phase 32 |
|-------|-------|-------------------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Compact row spacing within CollapsibleCard content |
| md | 16px | Card body padding, stat grid gap |
| lg | 24px | Section vertical rhythm (mt-6 between dashboard sections) |
| xl | 32px | Page-level top padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- CollapsibleCard header: `px-5 py-3` (20px / 12px) — inherited from existing `collapsible-card.tsx`, do not change.

- Touch targets for any interactive element (Quick Action buttons, nav items): minimum 44px height.

## Color

All values map to existing `globals.css` @theme tokens. No new colors introduced.

| Role | Token | Light value | Dark value | Usage |
|------|-------|-------------|------------|-------|
| Dominant (60%) | `background` | `#faf4ea` linen | `oklch(0.18 0.02 260)` | Page background, dashboard body |
| Secondary (30%) | `card` / `sidebar` | `#fffdf8` warm white | `oklch(0.20 0.02 258)` | CollapsibleCard surface, AppSidebar surface, MetricCard containers |
| Accent (10%) | `primary` = terracotta | `#c0673f` | `oklch(0.64 0.14 255)` | See reserved list below |
| Destructive | `destructive` | `#b23a2e` warm brick red | `oklch(0.55 0.20 25)` | Destructive actions only |

**Accent (`primary` / terracotta) reserved for:**
- MetricCard stat values with `color="blue"` variant uses `text-primary`
- Active nav item indicator in AppSidebar
- Ring focus state on interactive elements
- Quick Action CTA button (primary variant)
- SectionHeader title text (if using `text-primary` variant — match existing usage)

**Supporting tokens in use:**
- `text-muted-foreground` — stat card sublabels, secondary body text
- `text-text-dim` — MetricCard labels (`text-xs uppercase tracking-wider text-text-dim`)
- `success` / `warning` — status badges, health indicators (not new in this phase)
- `border` — CollapsibleCard header bottom border, card edges
- `sidebar-*` tokens — AppSidebar background, text, active item highlight

**60/30/10 distribution:**
- 60% `bg-background` (linen) fills the main dashboard content area
- 30% `bg-card` / `bg-sidebar` fills cards, CollapsibleCards, and the AppSidebar
- 10% `text-primary` / `bg-primary` accent appears on active nav, stat values, CTAs

## Dashboard Overview Layout Contract

The reworked `/dashboard/page.tsx` after Phase 32. This is the primary visual surface that changes.

### Stats Grid (DASH-02)

**Before (5 stats):** Posts · Photos · Albums · Events · Updates

**After (3 stats):** Photos · Albums · Events

- Layout: `grid grid-cols-2 @sm:grid-cols-3 gap-4` (drop `@lg:grid-cols-5`)
- Use `MetricCard` component with `color="default"` for all three, or match existing `<Card hover padding="md" className="text-center">` pattern — either is acceptable, but use one pattern consistently across all three stats
- Each stat links to its management area (`/dashboard/photos`, `/dashboard/photos/albums`, `/dashboard/events`)

### Quick Actions (DASH-02)

**Before:** Upload Photos · New Album · New Event · Quick Event · New Post · New Update

**After:** Upload Photos · New Album · New Event · Quick Event

- Remove "New Post" and "New Update" buttons entirely
- Remove `<QuickUpdateDialog />` component instance
- Keep `<QuickEventDialog />` component and its trigger button
- Keep Upload Photos link button and New Album link button

### CollapsibleCard Sections (DASH-02)

**Remove:** `<CollapsibleCard title="Recent Posts">` — deleted with no replacement

**Keep:** `<CollapsibleCard title="Upcoming Events">` — unchanged

**Add:** `<CollapsibleCard title="Recent Photos">` — shows the 4 most recently uploaded photos (thumbnail + filename/date). Query: `prisma.photo.findMany({ orderBy: { createdAt: "desc" }, take: 4, select: { id: true, thumbnailPath: true, createdAt: true, album: { select: { name: true } } } })`. Display: a 4-column thumbnail grid inside the card body using the `/api/images/[...path]` proxy. Empty state if no photos yet (see Copywriting Contract).

### Dashboard Sidebar Nav (DASH-01)

**Before:** Overview · Posts · Photos · Events · Updates · [owner: Members · Memorial]

**After:** Overview · Photos · Events · [owner: Members · Memorial]

- Remove `Posts` and `Updates` entries from `navLinks` in `src/app/(dashboard)/layout.tsx`
- Remove `Posts: FileText` and `Updates: Bell` entries from `iconMap` in `src/app/(dashboard)/app-sidebar.tsx` (removes dead `FileText` + `Bell` icon imports)
- Active item indicator: existing `sidebar-accent` / `sidebar-primary` tokens — do not change
