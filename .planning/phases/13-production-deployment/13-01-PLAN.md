<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → GHCR | Build pipeline pushes images; only main branch triggers builds |
| GHCR → Flux ImagePolicy | Flux polls GHCR; image tag format (timestamp) controls which image is "latest" |
| secrets namespace → homepage namespace | ExternalSecret copies secrets cross-namespace via ClusterSecretStore |
| Pod → external databases | JOBS_DATABASE_URL and DATABASE_URL are postgres:// connections to LAN hosts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | GHCR image tag | accept | Timestamp tags are monotonically increasing; only GitHub Actions (GITHUB_TOKEN) can push; no external write access |
| T-13-02 | Information Disclosure | hudsonfam-secrets in homepage ns | mitigate | ExternalSecret uses ClusterSecretStore with Owner creationPolicy — secret is owned by ExternalSecret, no manual kubectl create needed; secret is not logged |
| T-13-03 | Denial of Service | Resource quota blocks pod start | mitigate | Deployment uses Recreate strategy (not RollingUpdate); scale 0→1 fallback documented in Task 2 action |
| T-13-04 | Elevation of Privilege | Pod runs as root | mitigate | securityContext: runAsUser=1000, runAsNonRoot=true, readOnlyRootFilesystem=true, capabilities.drop=ALL — already in deployment.yaml |
| T-13-05 | Information Disclosure | JOBS_DATABASE_URL in pod env | accept | Env vars are only accessible inside the pod; no exec shell in prod (automountServiceAccountToken=false); secret sourced from ExternalSecret, not committed to git |
</threat_model>
