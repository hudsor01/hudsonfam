<objective>
Make `addPhotoToCollection` the single enforcement point for two locked rules:

1. COLL-01 — album exclusivity: a photo's "home" is at most ONE album-kind collection. When the target collection is kind `album`, atomically remove the photo's `CollectionPhoto` rows from every OTHER album-kind collection before (or within the same transaction as) creating the new membership. Surface-kind collections (`memorial`, `featured`) are references, not a home — adding to a surface collection must NOT remove the photo from its album, and album exclusivity must NOT touch surface memberships.

2. FEAT-04 — max-9 featured guard: the `featured` surface collection cannot exceed 9 photos. When the target collection is the `featured` surface collection, reject (throw) the add if it already holds 9. The existing `@@unique([collectionId, photoId])` already prevents duplicates — no new constraint needed.

Purpose: Every caller (dashboard grid, per-collection picker in Phase 39, featured manager in Phase 39) inherits both rules for free because they all funnel through this one action. No new action names are introduced (per CONTEXT specifics).

Output: Updated `src/lib/collection-actions.ts` and extended `src/__tests__/lib/collection-actions.test.ts`.

OUT OF PHASE (do NOT do here): seeding the `featured` row (Phase 40, gated live-DB), any UI (Phases 38/39), any Prisma schema change, any live-DB mutation.
</objective>

```ts
const EDIT_ROLES = ["owner", "admin", "member"];

export async function addPhotoToCollection(collectionId: string, photoId: string) {
  await requireRole(EDIT_ROLES);
  const sortOrder = await prisma.collectionPhoto.count({ where: { collectionId } });
  await prisma.collectionPhoto.create({ data: { collectionId, photoId, sortOrder } });
  await prisma.photo.update({ where: { id: photoId }, data: { published: true } });
  await revalidateSurfaces();
}

export async function removePhotoFromCollection(collectionId: string, photoId: string): Promise<void>;
export async function reorderCollectionPhoto(collectionId: string, photoIds: string[]): Promise<void>;
```
