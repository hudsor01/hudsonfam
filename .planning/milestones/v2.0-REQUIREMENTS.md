## v2.0 Requirements (all complete 2026-04-08)

### Phase 16: useEffect Audit

- [x] **EFFECT-01**: No useEffect derives state from props or other state (use useMemo or inline calculation)
- [x] **EFFECT-02**: No useEffect adjusts/resets state on prop change (use key prop or render-time adjustment)
- [x] **EFFECT-03**: No chained useEffects that trigger each other (consolidated into event handlers)
- [x] **EFFECT-04**: No useEffect for parent notification (moved to event handlers)
- [x] **EFFECT-05**: No useEffect for POST requests (moved to event handlers)
- [x] **EFFECT-06**: No useEffect for shared event logic (moved to event handlers)
- [x] **EFFECT-07**: Every remaining useEffect has proper cleanup
- [x] **EFFECT-08**: Every remaining useEffect synchronizes with a genuine external system

### Phase 17: Component Structure & State Patterns

- [x] **COMP-01**: No component is defined inside another component
- [x] **COMP-02**: All state updates create new object/array references
- [x] **BOUNDARY-01**: "use client" is at the lowest possible leaf component
- [x] **BOUNDARY-02**: No non-serializable props cross the server/client boundary
- [x] **BOUNDARY-03**: Data fetching happens in server components, not client useEffect/SWR
- [x] **BOUNDARY-04**: No prop-drilling through 3+ layers without composition
- [x] **BOUNDARY-05**: Server-only modules guarded with `import "server-only"` where applicable

### Phase 18: Server/Client Boundaries & Hydration

- [x] **RESILIENCE-01**: Every route group has loading.tsx
- [x] **RESILIENCE-02**: Every route group has error.tsx
