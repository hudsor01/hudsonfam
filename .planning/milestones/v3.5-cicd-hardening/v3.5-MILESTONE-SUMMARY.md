# v3.5 — CI/CD Hardening Milestone Summary

**Status:** CLOSED 2026-04-25
**Duration:** opened 2026-04-23 → closed 2026-04-25 (~3 days, 4 phases)
**Outcome:** SEED-005 thesis fully executed. The "every time I use it I have to fix it" DX pattern is permanently retired. hudsonfam deploys via the CLAUDE.md-intended pipeline (GitHub Actions + GHCR + Flux + K3s) — vendor-managed where possible (GitHub Actions + GHCR), self-hosted only where it must be (Flux + K3s).

## Why This Milestone Existed

Owner observation 2026-04-22: *"every time I use it I have to fix it which is a very low dx"*. Pre-v3.5 deploy pipeline had **6 moving parts, 5 of which were self-hosted**:

1. Forgejo repo (self-hosted; `forgejo-admin/hudsonfam` was deleted/missing at Phase 21 ship time)
2. Forgejo SSH endpoint (NodePort 30022)
3. Woodpecker server (5 restarts in 3 days during Phase 21 window)
4. Woodpecker agents + runner (3 pods)
5. `git.homelab` container registry (self-hosted, shared Longhorn volume)
6. Flux image automation (local; agnostic)

CLAUDE.md's **documented intent** was a 2-moving-part pipeline (GitHub Actions + GHCR — both vendor-managed). Reality had drifted — `.github/workflows/` didn't exist in hudsonfam; someone pivoted to Woodpecker at some point and the documented intent was orphaned. v3.0 Phase 21 production UAT exposed the rot when 77 commits couldn't reach production. v3.5 was scoped to fix this permanently.
